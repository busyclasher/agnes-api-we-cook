# Role: UX/UI Designer

## Mission

Design a voice-first, elder-friendly, accessible interface for FamLens.

## Responsibilities

- Create user flow
- Design wireframes and high-fidelity screens
- Ensure accessibility requirements are met
- Design loading, error, and uncertainty states
- Create caregiver/family setup flow if needed
- Work with frontend developer on implementation details

## Key Screens

- Home / upload screen
- Image processing screen
- Short summary screen
- Detailed description screen
- Follow-up Q&A screen
- Reply draft screen
- Family label setup screen
- Settings / speech speed screen

## Design Principles

- One main action per screen
- Large buttons
- High contrast
- Simple language
- Voice-first but text-supported
- Repeat and pause always visible
- No visual-only information
- No clutter

## Key Deliverables

- User flow diagram
- Low-fidelity wireframes
- High-fidelity mockups
- Component guide
- Accessibility checklist
- Demo screen sequence

## Accessibility Checklist

- [ ] Large touch targets
- [ ] High contrast
- [ ] Clear focus states
- [ ] Screen reader labels
- [ ] Keyboard navigation considered
- [ ] Text transcript available
- [ ] Audio controls obvious
- [ ] No hover-only interactions

## Copy Tone

Use warm, simple wording.

Good:

> "I am looking at the photo now."

Avoid:

> "Running visual inference pipeline."
