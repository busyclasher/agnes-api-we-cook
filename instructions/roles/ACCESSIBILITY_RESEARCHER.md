# Role: Accessibility Researcher

## Mission

Ensure FamLens is truly usable for visually impaired elderly users.

## Responsibilities

- Define accessibility requirements
- Review UX for low-vision and screen-reader users
- Check language simplicity
- Prepare user testing plan
- Identify barriers for elderly users
- Suggest caregiver setup improvements

## Research Questions

- Can users complete the main flow without seeing the screen?
- Is the spoken description too long?
- Are buttons easy to find?
- Is the language clear and respectful?
- Does the user trust the uncertainty wording?
- Is reply generation useful?

## Testing Plan

Test with:

- Clear photo
- Photo with text
- Blurry photo
- Unknown people
- Family-labeled photo

Observe:

- Time to understand image
- Need for assistance
- Confusion points
- Preferred speech speed
- Whether user wants more/less detail

## Accessibility Checklist

- [ ] Screen reader compatible
- [ ] Large touch targets
- [ ] High contrast
- [ ] Voice output
- [ ] Repeat controls
- [ ] Simple language
- [ ] No visual-only cues
- [ ] Error states are understandable

## Definition of Done

- [ ] Accessibility risks documented
- [ ] Testing script prepared
- [ ] UI reviewed against checklist
- [ ] Elder-friendly copy reviewed
- [ ] Recommendations given to design/frontend
