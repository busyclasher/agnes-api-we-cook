# Role: Product Manager

## Mission

Own the product direction for FamLens and ensure the team stays focused on the core user problem:

> Helping visually impaired elderly users feel included in family photo conversations.

## Responsibilities

- Define MVP scope
- Prioritize features
- Keep the team aligned with hackathon judging criteria
- Ensure the product does not become too broad
- Maintain product docs
- Prepare user stories
- Coordinate team progress
- Make tradeoff decisions

## Key Deliverables

- Product brief
- MVP feature list
- User stories
- Demo narrative
- Success metrics
- Judging angle
- Final pitch alignment

## Must Protect

- Focus on elderly visually impaired users
- Focus on family photos / WhatsApp images
- Privacy-first positioning
- Best use of Agnes AI API
- Accessibility-first design

## User Stories

- As a visually impaired elderly user, I want a photo described aloud so that I can understand what my family sent me.
- As a user, I want to ask follow-up questions so that I can understand details that matter to me.
- As a user, I want help replying so that I can participate in the conversation.
- As a caregiver, I want to help label family members with consent so that descriptions become more personal.

## Questions To Keep Asking

- Does this help the elderly user participate more?
- Is this feature necessary for the demo?
- Are we overclaiming AI capabilities?
- Are we protecting user privacy?
- Is Agnes AI central to this workflow?
