# Product Brief

## Project Name

**FamLens**

## One-Liner

FamLens helps visually impaired elderly users stay emotionally included in family photo conversations by turning images into spoken, personal, and conversational descriptions.

## Problem

Many elderly users receive family photos through WhatsApp and messaging apps, but visually impaired users may not be able to fully understand or respond to those images.

Existing accessibility tools can describe images, but descriptions are often generic. They may not explain emotional context, family relevance, visible text, or help the user reply.

## Target Users

### Primary Users

- Visually impaired elderly users
- Elderly users with low vision
- Elderly users who rely on WhatsApp or family messaging apps

### Secondary Users

- Family members
- Caregivers
- Community support workers
- Accessibility volunteers

## User Needs

Users need to:

- Understand what is in a photo
- Hear the description aloud
- Ask follow-up questions
- Understand visible text
- Know who may be in the photo
- Understand the mood or meaning of the moment
- Reply warmly to family

## Product Promise

When a user receives a family image, FamLens turns it into a spoken explanation that is simple, personal, and emotionally meaningful.

## MVP Scope

### Must Have

- Upload/share image
- Short spoken description
- Detailed description option
- OCR text reading
- Follow-up Q&A
- Reply draft generation
- Confidence and uncertainty wording
- Accessibility-friendly UI

### Should Have

- Family labeling setup
- Saved relationship context
- Multi-language support
- Caregiver setup flow
- Audio speed control

### Could Have

- WhatsApp share-sheet integration
- Memory timeline
- Automatic photo grouping
- Voice cloning with family consent
- Offline mode

### Out of Scope for MVP

- Automatic WhatsApp message sending
- Unconsented face recognition
- Medical diagnosis
- Public social-media posting
- Full photo album management
- Surveillance or live monitoring

## Differentiator

The differentiator is not **image captioning**.

The differentiator is:

- Family-context-aware descriptions
- Elder-friendly voice-first interaction
- Emotional understanding
- Follow-up conversation
- Reply assistance
- Honest uncertainty
- Privacy-first family setup

## Success Metrics

| Metric | Target | Why It Matters |
|---|---:|---|
| Time to first spoken summary | Under 10 seconds in demo | Keeps user experience smooth |
| User can understand image gist | 90% in user testing | Measures core value |
| Follow-up answer relevance | 80% useful answers | Measures conversational quality |
| Reply acceptance rate | 60% of generated replies accepted/edited | Measures participation value |
| Accessibility task completion | 90% can complete without visual guidance | Measures inclusive design |

## Product Risks

- Incorrect person identification
- Overly long descriptions
- Privacy concerns around family photos
- Elderly users struggling with setup
- AI hallucinating details
- Overpromising compared with existing accessibility tools

## Risk Mitigation

- Use uncertainty wording
- Keep summaries short first
- Do not store photos by default
- Make family labels opt-in
- Let caregivers help with setup
- Use confirmation before replies
- Clearly show confidence/fallback states
